## Perubahan Permanen yang TIDAK Boleh Dibalik

### 1. Google Drive UI dihapus dari Case Storage
- Tombol "Save Case to Google Drive" dan input "Google Drive Folder URL" **sengaja dihapus** dari UI Setup DB
- Alasan: berpotensi menyebabkan instabilitas aplikasi
- Backend `upload_case_archive_to_drive()` di `case_storage.py` tetap ada tapi tidak diekspos
- **Jangan re-add Google Drive UI ke Case Storage**

### 2. Format nama case ZIP
- Format baru: `porlungcase_{line_name}_{YYYYMMDD}_{HHMMSS}.zip`
- Digenerate otomatis dari `line_param.line_name` — tidak ada input manual case name lagi
- **Jangan ubah format ini atau kembalikan input manual case name**

### 3. Konstanta OHM
- `OHM = chr(0x03A9)` di `app_helpers.py` — pure ASCII, tidak bisa corrupt encoding
- **Jangan ubah ke literal karakter `Ω`** — akan menyebabkan mojibake `Î©` saat file di-save ulang
- Dipakai di `app.py` sebagai `{OHM}` di f-string metrik HR dan SE

### 4. OpenWeather API key BOLEH masuk case ZIP
- Sebelumnya dikecualikan dari snapshot, sekarang **disertakan**
- `openweather_lightning_api_key` dihapus dari `CASE_STATE_EXCLUDE_KEYS`
- Prefix `summary_weather_lightning_openweather` dihapus dari `CASE_STATE_EXCLUDE_PREFIXES`
- **Jangan kembalikan ke exclude list**

### 5. Sidebar upload dalam expander (default tertutup)
- Kedua grup upload (Local End dan Remote End) dibungkus `st.sidebar.expander(expanded=False)`
- **Jangan ubah ke `expanded=True` atau hapus expander**

## Bug yang Sudah Diperbaiki — Jangan Kembalikan

### Case ZIP restore infinite loop
- **Root cause:** Setiap `st.rerun()` membuat sidebar re-eksekusi dan `case_archive_file is not None` selalu True → loop tanpa henti
- **Fix:** MD5 hash file disimpan ke `st.session_state["_restored_case_hash"]`; restore hanya jalan jika hash berbeda
- Kode di `app.py` sekitar baris sidebar Case Storage
- **Jangan hapus pengecekan `_restored_case_hash`**

### Tower Map fault source tidak reset ke DE
- **Root cause:** Saat DE belum dihitung, `{key_prefix}_fault_source = "se_local"` tersimpan di session_state. Setelah DE dihitung, session_state tetap "se_local"
- **Fix:** Fungsi `_sync_fault_source_default(key_prefix, option_keys)` di `tower_map.py` — reset ke "de" saat DE pertama kali tersedia dalam sesi
- **Jangan hapus pemanggilan `_sync_fault_source_default` di `render_tower_map` dan `get_selected_fault_location_option`**

## Fitur Baru yang Ditambahkan

### CASE_SETTINGS_KEYS di case_storage.py
- Frozenset eksplisit berisi 36+ kunci yang selalu disimpan ke case ZIP
- Mencakup: DB URLs, sheet names, signal assignment widget keys, transformer data, line_param, OpenWeather key
- `_CASE_SETTINGS_WIDGET_FALLBACK` — sync widget input keys saat restore dari case lama
- `build_case_state_snapshot()` memiliki loop tambahan setelah loop utama untuk memastikan CASE_SETTINGS_KEYS selalu masuk

### Tabel tower sekitar fault — badge proteksi
- Fungsi baru di `tower_map.py`: `_protection_badge()`, `_build_protection_badges()`, `render_nearby_tower_table_html()`
- Menggantikan `st.dataframe` dengan `st.html` berisi tabel HTML kustom
- Kolom yang ditampilkan: No, SPAN, TYPE STRING, JUMLAH STRING, Proteksi Terpasang
- Kolom yang disembunyikan: Fault Context, Distance from Fault km, JARAK, KUMULATIF, LATITUDE, LONGITUDE, SEGMENT, ULTG
- Skema warna badge:
  - Hijau `#16a34a`: Cleaning Isolator
  - Kuning `#d97706`: Proteksi Petir DAN sub-devicenya (DGS, MGGS, TLA/NGLA, EGLA, Sumur Bor, MDG, MRG, DG, DMRG-A/B/C) — semua dalam satu kategori karena di spreadsheet mereka adalah sub-kolom dari PROTEKSI PETIR
  - Coklat `#92400e`: Dinding Penahan Tanah (Din. Penahan, Balok Kopel, Bronjong, Batu Kali, Sheet Pile, Shotcrete)
  - Merah `#dc2626`: Kerawanan Binatang (⚠ Burung/Kera/Ular)
  - Ungu `#7c3aed`: Proteksi Binatang (Jaring, Togar Abes, Kawat Duri, Terasi Kapur)
- Badge induk (misal "Proteksi Petir") **hanya muncul jika tidak ada satu pun sub-device yang terisi** — jangan ubah logika `elif` ini
- Badge menampilkan tanggal pemasangan di baris kecil bawah jika tersedia; "N/A", "nan", "0", "-" tidak ditampilkan
- Baris Before/After fault span: background `#fee2e2`; Closest tower: `#fef3c7`
